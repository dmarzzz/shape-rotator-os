Shape Rotator OS / information rules / reviewed 2026-06-14

Store information by authority, not convenience.

When something arrives, do not ask where it landed. Ask what the safest authoritative version is, who may see it, and which system owns the next step.

🔒 Raw source Drive vault or private local only.

Original transcripts, recordings, private notes, and operator evidence stay out of the public repo.

🧠 Distillation Supabase is the system of record.

Processing jobs, derived artifacts, reviews, approval gates, and app-safe views live in Supabase.

📚 Durable knowledge Repo data and docs are reviewed surfaces.

Commit only safe anchors, canonical cohort records, reviewed artifacts, policy, code, and generated bundles by intent.

The storage boundary

Drive captures. Supabase distills. Repo publishes reviewed truth.

This is the highest-value distinction in the index. It prevents raw material from leaking into public surfaces and prevents Drive folders from becoming the unofficial review database.

Drive vaultT0 source

Use for recordings, raw transcripts, source file names, and operator routing evidence.

  • Raw transcript stays in the room.
  • Calendar match before final route.
  • Run scripts/audit-transcript-labels.mjs for label and order review.
  • operator_review_exports is only an operator export mirror.
Supabasedistillation

Use for canonical processing and review state.

  • source_artifacts
  • processing_jobs
  • derived_artifacts, reviews, approval gates, app-safe views
  • Type, group, understanding, and card confidence percentages
Repositoryreviewed

Use for durable safe records, policy, code, and generated app bundles.

  • No raw transcript text.
  • No private exports.
  • Machine-enforced policy goes in cohort-data/policies/.

Privacy and publication tiers

Raw content never leaves T0/T1 control.

Public and cohort surfaces receive distillation, never raw transcript text. Attribution and quotes require explicit clearance.

People who were there. Raw transcript lives here only.

🛂 T1 Corerequest

Core team, coordinators, and project owners. Raw only by request and approval.

🧠 T2 Cohortdistilled

Gated cohort surface. Distilled readout with provenance, confidence, and review status.

🌐 T3 Publiccleared

Open public site. Generalized or explicitly cleared material only.

Promotion lifecycle

Raw input becomes durable OS knowledge only after review.

The workflow is intentionally slower than dumping files into the repo. The point is to preserve useful signal without widening private context.

📥01Inbox

Put unclassified arrivals in inbox, tmp/, or .private.

📅02Classify

Match calendar first. Identify type, source, date, audience, sensitivity, confidence percentages, and transcript label/order audit status.

📌03Anchor

Create safe metadata in events, Supabase source metadata, or a manifest.

🧠04Distill

Store canonical distillation state in Supabase. Ship T2 fast lane when allowed.

✅05Review

Mark generated output; below 70% stays review-held unless a human overrides.

🌐06Publish

Update canonical records, rebuild generated app bundles, and refresh INFORMATION_INDEX.html.

🚫07Archive or hold

Move private or blocked material to the correct private/held location.

Placement decision tree

Answer these questions in order.

The tree is intentionally blunt. If a file contains raw private content, no later answer can make it public-repo safe.

🔒
Does it contain raw private content?
Drive vault or cohort-data/.private/
📌
Is it a stable fact about a cohort object?
cohort-data/<record-type>/
📚
Is it a rule, plan, audit, architecture note, or process guide?
docs/
⚙️
Is it code or automation?
apps/, packages/, scripts/, or supabase/
🏗️
Is it rebuildable output or local scratch?
generated/, app bundle output, ignored build output, or tmp/
🌐
Is it a durable public article or recap?
cohort-data/articles/; rendered web copy goes where the app expects it.
Folder map

Use the folder that matches authority.

Folder choice should encode the source of truth. It should not merely reflect where a file happened to arrive.

📌cohort-data/

Canonical cohort knowledge and app-rendered markdown/data.

🔒cohort-data/.private/

Never commit. Vault plans, private exports, local-only review manifests.

📅cohort-data/events/

Calendar/event anchors. Use YYYY-MM-DD-short-event-name.md.

🧠cohort-data/session-readouts/

Reviewed session-level readouts. No raw transcript text.

⚖️cohort-data/policies/

Machine-readable JSON/YAML rules that scripts enforce.

📚docs/

Durable human-readable playbooks, audits, launch checklists, architecture notes.

⚙️apps/ and packages/

Application and shared package code. Do not store canonical cohort facts here except generated bundles.

🧹tmp/

Disposable local work. Anything here may be deleted.

Before promoting information

Run the checklist, then commit.

Most mistakes come from skipping the boundary check, not from bad filenames. Use this as the final gate.

✅ Review checklist

  • The file is in the right authority layer.
  • The filename follows the correct pattern.
  • Dates use YYYY-MM-DD.
  • Transcript date, type, and project were checked against the calendar before routing.
  • Transcript label/order review ran scripts/audit-transcript-labels.mjs or remaining content gaps are reported.
  • Transcript/source records include type, group, and understanding confidence percentages.
  • Public/cohort-visible material contains no raw private transcript text.
  • Derived material has provenance, confidence, review status, and audience boundary.
  • docs/INFORMATION_INDEX.html was regenerated after any transcript source, route, or review queue change.
  • Script-enforced rules live in schema or policy, not only prose.

🚫 Do not do this

  • Do not use the repository root as a dumping ground.
  • Do not store raw transcripts in docs/, apps/, public web folders, or session readouts.
  • Do not hand-edit generated files unless the file explicitly says it is hand-maintained.
  • Do not create parallel person/team files because the name changed.
  • Do not publish "transcript says" claims when the artifact is a paraphrased reconstruction.
