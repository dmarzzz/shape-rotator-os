Shape Rotator OS:
from cohort directory to cohort nervous system.

A local-first desktop client for a 10-week, in-person research cohort. The surface area is genuinely impressive — a P2P search atlas, NDI broadcast, an agent swarm, a PMF-journey graph, calendar, and a GitHub-backed profile editor. The gap isn't features. It's feedback loops, graceful failure, and the daily-open reasons that turn a beautiful directory into the place the cohort actually lives.

54 people · 26 teams self-scored 6/10 · "onboarding still rough" asks board: 0 real posts surfaces reviewed: 5 tabs · 11 OS modes

The core diagnosis

The product is read-heavy and pull-only. You can see everything and act on almost nothing without leaving for GitHub or a DM. Every social primitive — asks, profiles, intros — is a one-way launcher with no return path, no notification, no "did it work?"

The silent-failure tax

Half the app (atlas, network, metrics, swarm) dies quietly when the local swf-node daemon isn't running. Only the swarm panel fails gracefully. For a non-dev cohort member, "nothing is here" reads as "broken," not "start the daemon."

The tell

There are 54 people and 26 teams in the data, but the asks board — the single most important collaboration primitive in a cohort — has zero real posts (one example, one your-slug placeholder). That's not an adoption problem to wait out; it's a friction-and-feedback design problem. The same root cause throttles profiles, intros, and onboarding. Fix the loop and the data follows.

PART I

10 highest-impact bets

Larger moves that change what the product is for. Ordered roughly by leverage. Effort tags are engineering-weeks-class estimates, not promises.

01

Close the asks loop — make pairing two-way

flagshipeffort: medium
Today
An ask is a markdown PR that renders a card whose only action opens the author's DM. No claim UI, no "who responded," and a silent 5-day expiry. The author edits the file by hand to mark it claimed. Result: 0 real asks posted.
The bet
An in-app respond → claim → done lifecycle. "I'll pair" notifies the author, surfaces claimed_by, and routes responses as signed envelopes through swf-node / Matrix so it never touches a GitHub PR. Expiry becomes a visible countdown with a repost nudge.
Why
This is the social graph the whole cohort exists to create. It's half-built and untouched — the single highest-leverage fix in the app.
alchemy.js asks mode · schema has unused claimed_by · cohort-data/asks contains only example + placeholder
02

A presence & notification layer — "what changed since you looked"

retention drivereffort: medium
Today
Everything is pull. There is no reason to open the app on any given morning unless you already decided to. Live-peer state, new asks, calendar events, merged PRs — all sit there silently until you go find them.
The bet
A unified notification center + native OS notifications + a launch digest: "3 peers live · 2 new asks in your skills · standup in 15 min · your profile PR merged." Opt-in, quiet by default.
Why
This is the difference between a tool you remember exists and a tool you open daily. For a 10-week program, daily-open is the whole game.
glance/membrane already compute live-peer + ask + event counts — the data exists; only the push layer is missing
03

Daemon-optional by design + one-click bootstrap

reliabilityeffort: medium
Today
No swf-node → atlas blank, network "quiet," metrics empty, swarm pages vanish — mostly with no error. The research-agent also isn't bundled in production; users must git clone + uv sync by hand.
The bet
Bundle the daemon + research-agent in the installers. Everywhere that needs the daemon shows the same "daemon is down → [Start it] / [Install]" card (the atlas already has the pattern — promote it to a shared component). Add a one-screen health dashboard.
Why
A non-dev cohort member's first impression of three whole tabs is currently "empty / broken." This is table stakes for the install actually working.
swarm-node.js TODO: "production builds don't bundle a Python venv" · atlas-offline card exists, network/swarm have no equivalent
04

Lightweight, explainable matchmaking & smart intros

activationeffort: high
Today
Discovery is keyword filter + browse, by deliberate choice (no algorithms, no leaderboards). But the data is rich: skill_areas, seeking/offering, PMF journey.stage, prior work — all unused for connection.
The bet
A weekly, opt-in "3 people you should meet · 2 teams that should talk," driven by seek↔offer complementarity and skill overlap. Suggestion-only and explainable ("you both want help with attestation") — never a score or a rank. Honors the minimal-by-design ethos.
Why
In a cohort, the value is the introductions that wouldn't have happened. The data is already structured for it.
collab board computes seek↔offer overlaps already · schema skill_areas controlled vocab (~22 terms)
05

Edit your profile without leaving the app

unblocks funneleffort: medium
Today
The profile editor hands you off to GitHub: an unauthenticated fork-existence check, a blocking "create a fork" modal, a tab switch, a blind commit, a 24h cache that can re-prompt, and a 60 req/hr rate limit. It's also onboarding step 1 — so it gates everything.
The bet
In-app GitHub device-flow OAuth to commit directly, or route the edit as a signed envelope through swf-node and let a bot open the PR. Add a live markdown preview + diff before submit.
Why
The highest-friction step is the first step. Lowering it lifts every downstream number — including the empty asks board.
gh-fork.js fork-check modal · no OAuth · no in-app preview · profile-form.js inline (not yet shared with web)
06

Turn "context" into the cohort's institutional memory

differentiatoreffort: high
Today
13 session transcripts sit as raw .txt in a context tab with a manual "refresh index" button. The agent swarm is a separate box that researches the public web. The two never meet.
The bet
Point the swarm at the transcripts: ask "what did we decide about dstack attestation?", auto-generate per-session summaries / decisions / action-items, and link insights back to the people and teams who said them.
Why
Searchable institutional memory is a killer, genuinely-defensible feature — and every piece (swarm, transcripts, local index) already exists, just disconnected.
13 transcripts in src/content/context/raw-scripts · swarm-node.js routes through swf-node index · context mode is read-only
07

Ship the mobile companion (presence · calendar · asks · notify)

engagementeffort: high
Today
Tauri mobile is already scaffolded — renderer, prefs, remote swf-node over HTTPS all work; only the desktop-only subsystems are gated off. It's a read-only slice waiting to ship.
The bet
A focused phone app: who's in the building today, what's on the calendar, new asks, and the notification layer from bet #02. The desktop is for deep work; the phone is for "ping me when it matters."
Why
This is an in-person cohort at a physical school. The phone is where presence and coordination actually happen.
TAURI-MOBILE.md: renderer + remote swf-node already supported on iOS/Android · subsystems cleanly #[cfg(desktop)]-gated
08

Make the PMF journey a longitudinal coaching surface

program valueeffort: medium
Today
The journey data (ICP, problem, solution, evidence_quality, stage 1–8, milestones) renders as a static scatterplot inside constellation. It captures the actual point of a startup cohort but shows no movement.
The bet
A per-team trajectory over the 10 weeks: weekly check-in prompts, milestone tracking, "you said you'd reach X by week 4." Wire it to the encrypted depth/facilitator tier as the operator's program-management cockpit.
Why
It's the operator's reason to mandate the app, and the team's reason to care about their own record beyond a static card.
schema journey{stage, evidence_quality, milestones} · constellation "journey" sub-tab · depth tier already encrypted to alchemists
09

Easel → the cohort "stage" for demos & shared presence

in-personeffort: medium
Today
Easel is a polished but isolated NDI screen-broadcaster. You go live; people have to know to open easel and look.
The bet
A live-stream lobby + "now presenting" pinned into membrane/glance, a demo-day mode, and optional recording. One tap to "go live to the room," and the whole cohort sees it surface where they already are.
Why
Project intros and demos are central program rituals. Turning a hidden tool into ambient shared presence is high-value for a room full of people.
easel.js dual-canvas broadcast+receive is solid · no surfacing of "live now" into the default views
10

Make verifiable identity & trust real — dogfood the thesis

narrativeeffort: high
Today
This is a TEE / attestation / crypto cohort — yet identity is a self-asserted GitHub handle in a markdown field, and the computed making_signature ("code-verified" vs "stated") is barely surfaced.
The bet
Per-peer keypairs, signed profile/ask envelopes, sign-in-with-GitHub verification, and visible trust state — "this peer actually shipped the work they claim." Lean into attestation as a product surface, not just a topic.
Why
It's both real product value (anti-impersonation, provenance) and the most on-brand possible story: the cohort's app embodies the cohort's research.
making_signature{class:code-verified} computed but under-used · swf-node already signs page bundles · depth tier is encrypted
PART II

10 easiest UX wins

Mostly renderer-only, hours-to-a-day each, no architecture changes. These remove confusion and dead-ends that are actively costing the product credibility right now.

1

"Daemon offline" banner everywhere

Network, metrics, and the swarm silently show nothing when swf-node is down. Reuse the atlas's existing offline card as a shared component across all three. Kills the #1 "is this broken?" moment.

atlas-offline card → promote to shared · network/swarm reuse
2

Normalize skill tags + autocomplete

tee, TEE, and trusted-execution are three separate filters today. Lowercase + trim + map to the ~22-term controlled vocab on input, with a <datalist> in the profile and ask forms.

profile-form.js · asks compose · schema.yml vocab
3

Live preview before submitting a PR

Profile and ask edits are currently a blind commit — errors surface only after the PR opens. Render the record card from live form state right beside the form. Pure renderer work.

profile-form.js · alchemy.js asks compose
4

Pending state on async buttons

Easel "go live" (200–500 ms capture), CV scan, article-refresh, and swarm start all stay visually idle while working. Add a consistent spinner/label-swap pending state. The toast infra is already there.

easel.js:708 · alchemy.js CV scan · swarm form
5

Ask expiry countdown + repost nudge

Show "expires in 2 days" on each card (posted_at + 5d is right there), and toast the author with a one-tap repost when their own ask is about to vanish. Renderer-only.

alchemy.js asks mode (expiry already computed)
6

Replayable intro / "what is this"

The membrane intro fires once via localStorage with no way back. Add a quiet "?" replay affordance, plus a true first-run "what is this app" pass for new installs.

membrane intro modal · global first-run
7

Auto-clear stale atlas "new" halos

"New since last visit" halos persist across reloads until you manually click "clear new," so stale highlights mislead. Clear them on session start instead. Effectively a one-liner.

atlas.js "new since last visit" halos
8

Friendlier contact-link dead-end

When an ask author has no contact link, the card shows a dead disabled button. Swap it for "add a contact link →" pointing at the profile editor (your own) or an honest "no contact on file."

alchemy.js:~3524 contact fallback
9

Fix the once-per-session timeline auto-show

The network timeline auto-shows only on the first traffic event of a session, then ignores the user's later intent. Respect the toggle and re-show on first new traffic after a manual hide.

boot.js timeline _timelineAutoShown flag
10

Swarm trace: clear, cap, and warn-on-close

The trace buffer is unbounded (long runs get sluggish) and closing the modal silently leaves the agent running. Add a clear/copy button, cap the buffer, and toast "still running in the background" on close.

boot.js swarm trace append · panel close handler
if you only do three things

The sequenced bet

First
Make the install actually work — bet #03 + win #01
Bundle the daemon and add the offline banner everywhere. Nothing else matters if a fresh installer shows three empty tabs that read as broken. This is the floor.
Second
Close the asks loop + lower profile friction — bets #01 & #05
The two flows with proven zero/low adoption. Two-way pairing plus a frictionless first edit turn an empty directory into a living one. The data drought is a design symptom — fix the loop and content appears.
Third
Give people a reason to open it daily — bet #02
Presence + notifications convert a tool-you-remember into a tool-you-live-in. With the floor fixed and loops closed, this is what compounds engagement across the remaining weeks of the program.
